Apache Fluss Internal Architecture

CoordinatorServer + TabletServers — LogStore (Arrow IPC) + KvStore (RocksDB) with replication and tiering

Log Tables (Append-Only)

  • • Based on adapted Kafka replication
  • • Arrow IPC columnar format on disk
  • • LogTablet per bucket (replicated)
  • • Projection pushdown to filesystem
  • • Change types: +A (append), +I, +U, -U, -D

PK Tables (Mutable)

  • • RocksDB for efficient lookups
  • • Child LogTablet = changelog WAL
  • • Merge engines: None, FIRST_ROW, VERSIONED
  • • Partial update support
  • • Durability: changelog is replicated WAL

Tiered Storage

  • • Internal: log segments → S3 (tablet server)
  • • Lakehouse: data → Paimon/Iceberg (Flink job)
  • • Clients download tiered data directly
  • • Offloads tablet server load
  • • Future: bulk write support
