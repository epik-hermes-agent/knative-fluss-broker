Fluss Knative Broker

Durable event storage for Knative Eventing — powered by Apache Fluss

Fluss IS the Broker

  • Each Kubernetes namespace → Fluss database
  • Each Knative broker → Fluss log table
  • Arrow IPC columnar format on local disk
  • Adapted Kafka replication protocol
  • Sub-second write latency via RocksDB

Independent Trigger Cursors

  • Each trigger has its own scan cursor
  • Triggers don't compete for resources
  • Filter by CloudEvent type/source/attrs
  • Credit-based backpressure per lane
  • At-least-once delivery with retry

Credit-Based Backpressure

  • Each lane gets a token bucket of credits
  • Credits == 0 → pause scanning
  • On ACK → credit returns to bucket
  • Configurable refill rate per trigger
  • Slow subscriber only stalls its lane

Optional Iceberg Tiering

  • Fluss built-in — no custom code needed
  • Old segments → Parquet on S3/LocalStack
  • Polaris REST catalog
  • Tables opt in: datalake.enabled = true
  • Queryable by Flink, Spark, Trino
