Fluss + Iceberg Streaming Lakehouse

Three-tier storage: real-time (Fluss) → tiered (S3) → analytical (Iceberg) with transparent union read

Hot Tier — Fluss

  • • Arrow IPC columnar on local disk
  • • RocksDB for primary key lookups
  • • Sub-second read/write latency
  • • Adapted Kafka replication protocol
  • • Streaming reads via LogScanner

Warm Tier — Fluss Remote Storage

  • • Log segments uploaded to S3 by tablet server
  • • Client reads directly (offloads tablets)
  • • KvStore snapshots for recovery
  • • Configurable retention policy
  • • Automatic via RemoteLogManager

Cold Tier — Iceberg

  • • ACID transactions, time travel
  • • Schema evolution support
  • • Parquet columnar format
  • • Polaris REST catalog (Apache Polaris 1.3.0)
  • • Readable by Spark, Trino, Flink
